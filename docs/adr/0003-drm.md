# ADR 0003: VIP kontent uchun DRM (Bunny MediaCage) — hozircha YO'Q

- **Holat:** Qabul qilindi (Accepted)
- **Sana:** 2026-07-04
- **Vazifa:** P4-T3 (`drama_tasks.json`)

## Kontekst

Videolar Bunny Stream orqali uzatiladi. P4-T1/P4-T2 dan keyin himoya qatlamlari:

- **server-side gating** — free 1–10 / VIP / funding (`drama/services/playback.py`
  yagona manba, HTML + API bir xil);
- **imzolangan, muddatli URL** — token auth, default 4 soat, `token_path` papka
  imzosi, ixtiyoriy IP-binding;
- **referer allowlist** — hotlink/embed himoyasi (panel + `check_bunny_security`).

Savol: VIP kontentga qo'shimcha **DRM** (shifrlash + litsenziya nazorati) kerakmi?

Tahdid modeli ikkiga bo'linadi: **casual** yuklab oluvchi (yt-dlp, brauzer
kengaytmasi, link ulashish) va **determined** pirat (screen capture — buni hech
qanday DRM to'xtata olmaydi, "analog hole").

## Variantlar

1. **MediaCage Basic** (bepul toggle): faqat Bunny'ning **iframe** pleyeri bilan
   ishlaydi — custom hls.js pleyer bilan EMAS. Yoqilsa yo'qoladi: WatchProgress /
   davom ettirish (P1-T3), avtomatik keyingi qism, sifat boshqaruvi va P5-T2
   ilg'or pleyer rejalari. Himoyasi: casual ripping qiyinlashadi, xolos.
2. **Enterprise DRM** (Widevine/FairPlay): enterprise narx + EME-pleyer
   (Shaka/hls.js EME) + Apple FairPlay sertifikati + litsenziya-server
   murakkabligi. Widevine L3 baribir muntazam buziladi.
3. **DRM'siz** — joriy qatlamli himoya bilan qolish.

## Qaror

**DRM yoqilmaydi.** Joriy himoya (gating + muddatli imzo + referer) casual
piratlikka mutanosib; DRM'ning xarajati/murakkabligi foydasidan ustun.

## Sabablar

1. **Mahsulot zarari:** MediaCage Basic custom pleyerni yo'q qiladi — P1-T3
   (resume) ishlamay qoladi, P5-T2 (ilg'or pleyer) imkonsiz bo'ladi.
2. **Xarajat/foyda:** Enterprise DRM narxi va murakkabligi bozor hajmiga
   nisbatan asossiz; Widevine L3'ning o'zi ham ishonchli to'siq emas.
3. **Mutanosiblik:** determined piratni DRM ham to'xtatmaydi (screen capture);
   casual piratga esa joriy qatlamlar yetarli.
4. **Qaytarish oson:** qaror hech qanday kodni qulflamaydi — keyin yoqish yo'li
   ochiq (quyida yo'l xaritasi).

## Qayta ko'rish trigger'lari

- Kontent litsenziya beruvchisi (distribyutor) shartnomada DRM **talab qilsa**;
- tizimli restream/tarqatish aniqlansa (masalan, Telegram kanallarida) va
  token+referer qatlami yetmayotgani ko'rinsa;
- Bunny DRM taklifi o'zgarsa (custom-player bilan mos Widevine, maqbul narx).

## Agar keyin yoqilsa (yo'l xaritasi)

- Faqat VIP kontentga (`Movie.is_vip`); `drama/services/playback.py` allaqachon
  yagona chiqish nuqtasi — gating o'zgarmaydi, faqat qaytariladigan URL turi
  o'zgaradi (signed HLS o'rniga DRM litsenziyali manba yoki signed `embed_url`).
- **MediaCage Basic yo'li:** VIP'ga signed iframe embed, tekin qismlarga custom
  pleyer — aralash UX'ni ongli qabul qilish kerak bo'ladi.
- **Enterprise yo'li:** Shaka/hls.js EME pleyer + Bunny DRM sozlamasi + FairPlay
  sertifikat + litsenziya endpoint.

## Oqibatlar

- **Ijobiy:** pleyer erkinligi saqlanadi (P5-T2 uchun yo'l ochiq), qo'shimcha
  xarajat va operatsion murakkablik yo'q.
- **Salbiy (qabul qilingan risk):** texnik savodli foydalanuvchi imzoli URL amal
  qilish oynasida (≤4 soat) stream'ni yuklab olishi mumkin. Zaxira richaglar:
  `BUNNY_TOKEN_EXPIRY_SECONDS` ni qisqartirish, `BUNNY_TOKEN_BIND_IP` ni yoqish
  (docs/ops/bunny.md dagi ogohlantirish bilan).
